## Phase 3 — Create Cloud Run Jobs

- [x] **3.1** Create Cloud Run Job for **generation**:
  ```bash
  gcloud run jobs create content-generate \
    --image us-central1-docker.pkg.dev/agentic-learning-app-e13cb/agentic-learning/content-generate:latest \
    --region us-central1 \
    --service-account cloud-run-app-identity@agentic-learning-app-e13cb.iam.gserviceaccount.com \
    --set-env-vars GCS_PIPELINE_BUCKET=agentic-learning-pipeline \
    --memory 1Gi \
    --task-timeout 7200 \
    --max-retries 0
  ```
  `--max-retries 0` — the pipeline manages its own crash recovery via `--resume`.

- [x] **3.2** Create Cloud Run Job for **embedding** (same image, different entrypoint):
  ```bash

  --image us-central1-docker.pkg.dev/agentic-learning-app-e13cb/agentic-learning/content-generate:latest \
  --region us-central1 \
  --service-account cloud-run-app-identity@agentic-learning-app-e13cb.iam.gserviceaccount.com \
  --set-env-vars GCS_PIPELINE_BUCKET=agentic-learning-pipeline \
  --command python \
  --args="embed_content.py,--resume" \
  --memory 1Gi \
  --task-timeout 3600 \
  --max-retries 0
  ```

## Crash Recovery Summary

| What crashes | What survives | Recovery action |
|---|---|---|
| Generation job mid-run | `pipeline_log.json` + all written files in GCS | Re-execute job with `--resume` |
| Embed job mid-run | Approved files in GCS unchanged | Re-execute `content-embed` job (idempotent) |
| Seed job mid-run | Cloud SQL rows already written (idempotent) | Re-execute `content-seed` job; `ON CONFLICT` handles duplicates |
